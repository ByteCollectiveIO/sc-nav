ORG SETTINGS (admin)

Off by default — your org starts from a blank POI database (only your own custom POIs and QT markers). Check this to import the full starmap.space catalog right now (no restart needed); once imported it re-fetches the latest on every restart. Uncheck to remove it and go back to a clean database immediately — your custom POIs are kept. Celestial bodies are always loaded.

ACCESS

Require a Discord role (in addition to being in the server) to sign in. Leave blank to allow any member. In Discord with Developer Mode on, right-click the role → Copy Role ID and paste it here. Admins always have access; a change takes effect at each member's next sign-in.

ADMINS

Admins change these org settings and can delete any contribution. Add a member by their Discord user ID (Developer Mode on → right-click a user → Copy User ID); changes take effect immediately. Admins set in the server's environment show as root and can't be removed here.

OBSERVATION FRESHNESS

Mark sightings stale after hours
Resource nodes and fauna are ephemeral — SC respawns them. The map markers and NEARBY list hide sightings older than this (the map's "fresh only" toggle is the per-view escape hatch; the ore heatmap always uses every sighting). Tune this as you learn SC's respawn cadence.

SET UP THE WATCHER

Org Navigator shows where you are in the 'verse on this screen, but it needs a small helper — the watcher — running on your gaming PC. The watcher reads the /showlocation coordinates the game copies to your clipboard and sends them here. Set it up once, then forget it. The steps below take about five minutes.

  1. You're signed in ✓

    You reached this page through Discord, so this step is already done. Everything below happens on the gaming PC where you play Star Citizen.

  2. Download your watcher

    Click below to get a ready-to-run bundle. It's already pointed at your server (this server) and has your private access token built in — nothing to type, nothing to edit.

    Unzip it somewhere you'll find again, like your Desktop. You'll see a watcher folder with run_watcher.bat inside.

    The download contains a personal token tied to your account, so keep the file to yourself. You can revoke it any time under ⚙ Settings (handy if you re-download or change PCs).

  3. Install Python (one time)

    The watcher needs Python — a free program many apps use. Get it from python.org. In the installer, check the box that says "Add python.exe to PATH" before clicking Install. If Python is already on this PC, skip this step.

  4. Run it

    Double-click run_watcher.bat in the folder you unzipped. A black window opens and asks for your in-game handle — your RSI/Star Citizen name, so sightings you capture are credited to you. Type it and press Enter.

    That's the only thing it asks; your token is already built in. It remembers the handle, so next time you just double-click and play. Leave the window open (minimized is fine) while you play. When you see listening for /showlocation, it's working.

    Prefer the command line? 
    python sc_nav_watcher.py --server SERVER --handle "YourName"

    The token is read from the bundled watcher_config.json, so you don't pass it on the command line.

  5. Send your first position

    In Star Citizen, open the chat box (your chat key, Enter by default), type /showlocation and press Enter. The game copies your coordinates; the watcher sends them here within a fraction of a second.

    Come back to the navigation view — your position, bearing, and distance to your destination will be live. Re-run /showlocation whenever you want to update it (the up-arrow in chat re-sends it fast).

Manage or revoke your tokens any time under ⚙ Settings.
